01 Introduction

APTS is a web-based application designed to help athletes log, review, and analyze their training sessions over time. Athletes record data like distance, duration, altitude, and temperature; the system automatically calculates pace and speed, and visualizes trends using interactive charts.

Core Goals

  • Record detailed training sessions with per-kilometer splits
  • Calculate average speed, pace, and performance metrics automatically
  • Visualize progress over time with charts and dashboards
  • Secure each athlete's data with JWT authentication

Scope

The application covers three areas: User Management (register, login, profile), Session Tracking (create, view, edit, delete training records), and Analytics (trends, comparisons, summaries).

How a Request Flows

  1. User ActionAthlete clicks "Save Session" on the frontend form.
  2. Axios RequestThe frontend api.js client sends POST /api/sessions with a JWT in the Authorization header.
  3. MiddlewareExpress receives the request; the auth.js middleware verifies the JWT and attaches req.user.
  4. ControllerThe sessionsController receives the request, calculates avg_speed and avg_pace, then runs an SQL transaction to insert the session and all km splits.
  5. DatabasePostgreSQL stores the record; the connection pool returns the result rows.
  6. ResponseThe API returns a JSON response; the frontend updates state and redirects to the sessions list.

Table: kilometer_splits

idUUIDPKAuto-generated split ID
session_idUUIDFKReferences training_sessions.id — ON DELETE CASCADE
kilometer_numberINTEGERNOT NULL1, 2, 3 … N (km marker)

Key Design Decisions

  • UUID primary keys — harder to enumerate than integers, safer for APIs
  • Composite index on (user_id, training_date) — speeds up dashboard queries filtered by both
  • ON DELETE CASCADE — deleting a user removes all their sessions and splits automatically
  • Trigger: update_updated_at — automatically sets updated_at on row changes
  • avg_speed / avg_pace stored (not just calculated at query time) for faster analytics aggregation

06 Backend — Node.js + Express

Server Setup (server.js)

server.js is the entry point. It creates an Express app, registers global middleware (CORS, JSON body parser, request logger), mounts the three route groups, adds a 404 handler, and starts listening on the configured port.

server.js — simplified
const app = express();
app.use(cors({ origin: process.env.FRONTEND_URL }));
app.use(express.json());

app.use('/api/auth',      authRoutes);      // public + protected
app.use('/api/sessions',  sessionsRoutes);  // all protected
app.use('/api/analytics', analyticsRoutes); // all protected

app.listen(process.env.PORT || 5000);

Authentication Flow

Registration: The user submits name, email, and password. The controller checks for duplicate emails, hashes the password with bcrypt (10 salt rounds), stores the hash, and returns a JWT.

Login: The controller fetches the user by email, runs bcrypt.compare() against the stored hash. On match, it issues a JWT signed with JWT_SECRET (7-day expiry). The password is stripped before returning the user object.

Protection: The auth.js middleware reads the Authorization: Bearer <token> header, verifies the token with jwt.verify(), and attaches req.user = { id, email, name }. All session and analytics routes use this middleware.

Automatic Calculations

These are computed in sessionsController.js before inserting into the database:

controllers/sessionsController.js
// Average speed (km/h)
avg_speed = distance_km / (duration_minutes / 60)

// Average pace (min/km)
avg_pace  = duration_minutes / distance_km

// Per-km split speed (km/h)
split_speed = 3600 / split_time_seconds

Session inserts are wrapped in a database transaction (BEGIN / COMMIT / ROLLBACK) so that if any km split insert fails, the whole operation is rolled back cleanly.

Authentication Context (AuthContext.js)

A React Context wraps the entire app. It stores the current user object in state, reads the JWT from a cookie on first load, and provides login(), register(), and logout() functions to all child components.

Protected layouts check useAuth().user; if null after loading, they redirect to /auth/login using Next.js's useRouter().

API Client (lib/api.js)

An Axios instance configured with baseURL = NEXT_PUBLIC_API_URL/api. Two interceptors are registered:

  • Request interceptor: reads apts_token from cookies and injects Authorization: Bearer … into every request
  • Response interceptor: on a 401 response, clears the cookie and redirects to login

Design System

All styling lives in app/globals.css using native CSS custom properties (no Tailwind or styled-components). The palette uses a dark base with an electric lime-green accent (#c6f135) and secondary electric blue (#4facfe).

Typography: Syne (display/headings, geometric, bold) + DM Sans (body, clean) + DM Mono (code, numbers). Loaded from Google Fonts.

Component patterns: .stat-card, .card, .btn, .form-input, .badge, .nav-link — all defined as reusable CSS classes. Components use className strings to apply them.

09 Setup Guide

Prerequisites

  • Node.js v18 or newer
  • PostgreSQL 14 or newer
  • VS Code (recommended extensions: ESLint, Prettier, PostgreSQL)
  • npm (bundled with Node.js)

Step-by-Step

  1. Clone / Download the project Open the APTS folder in VS Code. You'll see three subfolders: database, backend, frontend.
  2. Create the PostgreSQL database Open a terminal and run: 
    psql -U postgres -c "CREATE DATABASE apts_db;"
psql -U postgres -d apts_db -f database/schema.sql
  3. Configure backend environment Copy .env.example to .env inside the backend folder and fill in your values: 
    DATABASE_URL=postgresql://postgres:YOUR_PASSWORD@localhost:5432/apts_db
JWT_SECRET=any_long_random_string_here
PORT=5000
FRONTEND_URL=http://localhost:3000
  4. Install backend dependencies 
    cd backend
npm install
  5. Start the backend 
    npm run dev    # uses nodemon for auto-restart
# Server running at http://localhost:5000
  6. Configure frontend environment Copy .env.local.example to .env.local inside the frontend folder: 
    NEXT_PUBLIC_API_URL=http://localhost:5000
  7. Install frontend dependencies 
    cd frontend
npm install
  8. Start the frontend 
    npm run dev
# App running at http://localhost:3000
  9. Open in browser Navigate to http://localhost:3000. You'll be redirected to the login page. Register a new account and start logging sessions!
